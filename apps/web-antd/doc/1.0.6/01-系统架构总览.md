# 01 · 系统架构总览

## 1. 产品定位

**Voglander** 是基于 Spring Boot 3.5.3 + Java 17 的企业级视频监控 / 流媒体平台，支持 **GB28181-2016 / GT1078 / ONVIF** 等协议，提供设备接入、实时直播、录像回放、PTZ 控制、流媒体编排（拉流/推流代理）、协议验证台等能力。前端 **web-antd**（Vue 3 + Ant Design Vue）是驱动后端的管理控制台。

## 2. 多仓库工作区

工作区 `voglander-workstation` 本身不是 git 仓库，包含四个**独立**仓库，共同构成一个产品：

| 仓库 | 角色 | 形态 |
| --- | --- | --- |
| `voglander/` | 产品集成方，企业视频监控平台 | 可运行应用（`voglander-web`） |
| `sip-proxy/` | GB28181-2016 SIP 框架 | Maven **库**（被嵌入） |
| `zlm-spring-boot-starter/` | ZLMediaKit REST/Hook 集成 + 负载均衡 | Maven **starter** |
| `vue-vben-admin/` | 管理前端（本仓库，目标 app = `apps/web-antd`） | Web 应用 |

### 依赖链（关键大局）

`voglander` 是集成者，把另外三者作为**版本化依赖**消费，三者位于 voglander 上游：

```
sip-proxy ──► sip-gateway-spring-boot-starter / gb28181-client / gb28181-server  (v1.8.0)
                                                  │
zlm-spring-boot-starter ──► zlm-spring-boot-starter  (v1.0.10-SNAPSHOT) ─────────┤
                                                                                 ▼
                                                          voglander (voglander-integration)
                                                                                 ▲
                          vue-vben-admin ──► REST  /api/v1/* , /auth/* , /user/* , /system/* , /zlm/api/*
```

**构建顺序含义**：voglander 依赖 zlm starter 的 `-SNAPSHOT`，改了 `sip-proxy` 或 `zlm-spring-boot-starter` 必须先 `mvn install` 进本地 `~/.m2`，voglander 才可见。仅改 voglander 无需重建上游。

## 4. 典型数据流

### 4.1 普通 CRUD（以拉流代理为例）

```
list.vue（VXE Grid proxyConfig.ajax.query）
  → getStreamProxyPageList()  POST /api/v1/proxy/getPage
  → vite 代理剥离 /api 前缀，转发 :8081 → StreamProxyController.getPage
  → StreamProxyManager.getPage(dto,page,size)（模板方法，createTime 降序）
  → AjaxResult.success({total, items})
  → requestClient 解包 data（successCode=0）→ Grid 渲染
```

### 4.2 ZLM 流媒体编排（跨系统）

```
前端 → POST /api/v1/proxy/createStreamProxy（落库业务记录）
前端 → POST /zlm/api/proxy/add（带 X-Node-Key 指定 ZLM 节点）
  → voglander 经 zlm starter 代理到目标 ZLM 服务器
  → ZLM 拉流成功 → 触发 Hook 回调 → VoglanderZlmHookServiceImpl 落库
  → ZLM 心跳 → Hook → 节点在线状态写库
```

### 4.3 GB28181 设备事件（实时推送）

```
设备 SIP 注册/心跳/目录/告警
  → sip-proxy 框架 → VoglanderBusinessNotifier.notify(GatewayEvent)（@Async）
  → 分发 DeviceRegisterService / DeviceManager / DeviceChannelManager / MediaSessionManager
  → RedisBackedSseEventBus 发布事件（event 名 = 完整 topic，如 device.register）
  → SseController /api/v1/stream/events 推送
  → 前端 useSseEvents EventSource 按 topic 分流渲染（protocol-lab 时间线）
```

## 5. 技术栈

### 后端（voglander）

- Java 17（强制 `jakarta.*`）、Spring Boot 3.5.3、MyBatis-Plus 3.5.5、Dynamic DataSource 4.3.1
- 持久化：SQLite（开发/测试，`app.db` 自动创建）/ MySQL 8.2.0（生产），HikariCP，Redis
- 序列化：**FastJSON2**（禁止 Jackson/Gson）
- 集成：sip-gateway 1.8.0、ZLMediaKit-Starter 1.0.10-SNAPSHOT、EasyExcel 4.0.1
- 文档/监控：SpringDoc 2.8.9、SkyWalking 9.1.0；测试 JUnit 5 + Mockito

### 前端（web-antd）

- Vue 3 + TypeScript + Composition API，Ant Design Vue
- pnpm workspace + Turborepo Monorepo
- 路由 Vue Router 4（后端驱动菜单/权限），状态 Pinia（持久化 + 加密）
- 表格 VXE Table，表单 Schema 驱动（`@vben/common-ui`），图标 `@vben/icons`（Lucide）
- 媒体播放 Video.js / FLV.js / HLS.js，样式 Tailwind CSS
- 构建 Vite（`@vben/vite-config` 自定义插件系统）

## 6. 部署与运行形态

| 维度 | 说明 |
| --- | --- |
| 后端入口 | `ApplicationWeb`（`mvn spring-boot:run -pl voglander-web`），默认端口 `8081` |
| 前端开发 | `pnpm dev:antd`（端口 `5666`），`VITE_GLOB_API_URL=/api` |
| 开发代理 | vite `server.proxy['/api']` → `http://0.0.0.0:8081`，rewrite 剥离 `/api` 前缀，`ws:true` |
| 配置 profile | `application.yml`(主)/`-dev`/`-test`/`-repo`(DB)/`-inte`(集成，含 SIP `gateway:` 段) |
| SIP 启动 | `ApplicationWeb` 必须 `@EnableSipServer`；`start/ServerStart` 绑定 SIP 端口（唯一 `addListeningPoint`） |
