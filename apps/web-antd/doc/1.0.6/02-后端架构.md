# 02 · 后端架构（voglander）

> 根包 `io.github.lunasaw.voglander`。版本 `1.0.2-SNAPSHOT`。
> 本文档面向「前端如何理解后端」，权威细节以 `voglander/CLAUDE.md` 为准。

## 1. 多模块结构与依赖

| 模块 | 职责 | 层 |
|------|------|----|
| `voglander-web` | REST 控制器、过滤器、拦截器；应用主模块，承载**所有测试** | Web |
| `voglander-manager` | 业务编排、复杂多服务协调（含 `MediaSessionManager`） | Manager |
| `voglander-service` | 核心业务服务，`extends IService<DO>`，单表单一职责 | Service |
| `voglander-repository` | 实体(DO)、Mapper、缓存 | Repository |
| `voglander-integration` | 外部系统集成（GB28181/ZLM/Excel），统一 `ResultDTO` | Integration |
| `voglander-client` | 外部服务客户端与 DTO | Client |
| `voglander-common` | 常量、枚举、异常、工具（含 `AjaxResult`/`ApiConstant`/`JwtUtils`） | Common |
| `voglander-test` | 测试配置与工具 | Test |
| `voglander-coverage-report` | JaCoCo 聚合报告 | — |

**依赖方向**（实测 pom）：
```
web → integration, service
service → client, common, integration, manager
manager → integration, repository
integration → client, common, manager
repository → client, common, (sqlite)
client → common
common → （无内部依赖，最底层）
```
总体：`web → manager → service → repository → common`；`integration / client → common`。

## 2. 分层职责

- **Web**：REST 控制器、参数校验、Req/VO 格式转换。**入参必须用 Web 层 `*Req`，不得直接收 DTO**；出参用 VO。
- **Manager**：复杂业务流编排、多服务协调。**对外方法参数/返回一律用 DTO，不暴露 DO**。
- **Service**：`extends IService<DO>`，单表核心��辑。
- **Repository**：持久化与缓存。
- **Integration**：外部系统包装器（Wrapper），统一 `ResultDTO` 响应。

### 分层调用选择
- **复杂业务**（多表/复杂逻辑）：`Controller → 专用 Service → 多个 Manager`，该 Service 也可直接调 `IService` 基础方法。
- **简单单表**：`Controller` 可直接调 `IService`，无需经 Manager。

## 3. Controller 模板方法规范

Controller 对应 Manager 标准 CRUD，按优先级实现：
- **必须（核心模板）**：`add` / `update` / `get` / `deleteOne` / `deleteBatch` / `getPage`
- **可选（增强业务）**：含操作日志的 `createXxx` / `updateXxx` / `deleteXxx`
- **避免（专用细粒度）**：状态更新、Key 更新等通常不在 Controller 暴露

关键约定：
- 入参：`*CreateReq` / `*UpdateReq`（含 `@NotNull` 的 ID）/ `*QueryReq`，经 `WebAssembler.xxxReqToDto()` 转 DTO
- 出参：Manager 返回 DTO → `WebAssembler.dtoToVo()` → 前端
- 更新：必带主键 ID，走 `manager.updateById(id, updateDTO)`
- 查询：实现**全量分页条件查询**接口，由前端灵活组装条件
- 分页包装为 `*ListResp`（含 `total` + `items`）
- 统一返回 `AjaxResult.success()` / `AjaxResult.error()`

```java
@PostMapping("/add")
public AjaxResult<Long> add(@Valid @RequestBody StreamProxyCreateReq req) {
    return AjaxResult.success(streamProxyManager.add(webAssembler.createReqToDto(req)));
}
```

### Manager 模板方法统一签名
```java
public Long add(XxxDTO dto)
public Long update(XxxDTO queryDTO, XxxDTO updateDTO)   // 查询条件与更新内容分离
public Long updateById(Long id, XxxDTO updateDTO)        // 最常用
public XxxDTO get(XxxDTO dto)
public Boolean deleteOne(XxxDTO dto)
public Boolean deleteBatch(XxxDTO dto)
public Page<XxxDTO> getPage(XxxDTO dto, int page, int size)  // 默认按 createTime 降序
private void clearCache(Long id, String oldKey, String newKey)  // 主键 + 业务键双清理
```

## 4. 统一响应契约 · AjaxResult

`voglander-common/.../domain/AjaxResult.java`，本质是 `HashMap<String,Object>`：

| 字段 | 含义 |
|------|------|
| `code` | 状态码：**`0`=成功**、`300`=警告、`500`=错误 |
| `msg` | 提示文案（默认「操作成功」/「操作失败」） |
| `data` | 数据本体（泛型 `T`，为 null 时不写入 map） |

```java
AjaxResult.success()              // {code:0, msg:"操作成功"}
AjaxResult.success(data)          // {code:0, msg:"操作成功", data:...}
AjaxResult.error("xxx")           // {code:500, msg:"xxx"}
AjaxResult.error(code, "xxx")     // 自定义码
```
> 前端 `requestClient` 据此配置 `successCode:0` / `dataField:'data'`，详见 [04 契约](./04-前后端集成与接口契约.md)。

## 5. 鉴权机制（JWT / Bearer）

- 登录 `POST /auth/login` → `AuthService.login()` 返回 access token → `LoginResp.accessToken`
- 后续请求头 `Authorization: Bearer <token>`，由 `JwtAuthInterceptor`（`voglander-web/interceptor`）校验
- token 解析工具 `JwtUtils`（`voglander-common`），`JwtUtils.getUserId(token)` 取用户 ID
- `POST /auth/refresh`：用旧 token 刷新新 token（`RefreshTokenResp`）
- `GET /auth/codes`：返回当前用户权限码列表 `List<String>`（前端 `hasAccessByCodes` 据此判定）
- `POST /auth/logout`：注销
- 内部 SIP 接口 `/internal/sip/*` 由 `InternalAuthFilter` 单独鉴权（非用户 token）
- 其他过滤器：`XssFilter`、`TraceFilter`（链路）、`RepeatableFilter`（可重复读 body）

## 6. 数据 / 时间 / JSON 规范（强制）

- **时间**：DO/DTO 层统一 `LocalDateTime`；**VO 返回 Unix 毫秒时间戳**，字段以 `Time` 结尾，DTO 提供 `xxxToEpochMilli()` 转换
- **JSON**：全项目 FastJSON2；类型转换须经正反序列化 `JSON.parseObject(JSON.toJSONString(src), Target.class)`
- **MyBatis-Plus**：简单 CRUD 用 `BaseMapper`/`IService`，禁自定义 SQL；仅复杂/联表/批量才写 SQL
- **缓存**：`@Cached`/`@Cacheable` 仅用于 Repository 单对象查询；复杂缓存用 `RedisCache`（FastJSON 字符串）；分布式锁 `RedisLockUtil`
- **异常**：抛 `ServiceException`（先确认 `ServiceExceptionEnum` 有对应类型）；全局 `GlobalExceptionHandler`
- **LambdaQueryWrapper**：必须用带 `condition` 参数的链式方法，禁止 `new LambdaQueryWrapper<>(entityObject)`

## 7. 外部系统集成

### 7.1 GB28181（sip-proxy，sip-gateway 1.8.0）
- 架构已从「handler 分发」迁移到「**command 出站 + 统一回调**」
- **出站指令**：`client/command/`、`server/command/` 按业务域分包（ptz/record/device/alarm/catalog/status/config/media），继承 `AbstractVoglanderClientCommand` / `AbstractVoglanderServerCommand`
- **入站回调唯一入口**：`notifier/VoglanderBusinessNotifier implements BusinessNotifier`，`notify(GatewayEvent)` 必须 `@Async("sipNotifierExecutor")`；事件三段式 `gb28181.Group.Name`，payload `Map<String,Object>` 用 FastJSON2 反序列化
- **会话状态**：`MediaSessionManager` + `MediaSessionDO`(`tb_media_session`)，业务主键 SIP `callId`，状态 ACTIVE=1/CLOSED=0/INVITING=2/FAILED=3
- **多节点**：`RedisInviteContextStore`（`sip:invite:ctx:{callId}`）支撑跨节点回包路由
- **启动**：`ApplicationWeb` 必须 `@EnableSipServer`；`start/ServerStart` 绑定 SIP 端口不可删除

### 7.2 ZLMediaKit（zlm starter 1.0.10-SNAPSHOT）
- 后端 `/zlm/api/*` 代理到 ZLM；`VoglanderZlmHookServiceImpl` 处理回调（服务生命周期、流事件、节点心跳/状态）
- 配置 `zlm:`（`enable`/`hook-enable`/`servers[]`），`ZlmIntegrationConfig#getDefaultServer()` 取默认节点
- **流代理流程**：前端 → `/zlm/api/proxy/add` → ZLM → Hook 回调 → 落库
- 多节点路由：前端通过 `X-Node-Key` 头指定目标 ZLM 节点（见 [04 契约](./04-前后端集成与接口契约.md)）

### 7.3 SSE 实时事件
- `SseController` `GET /api/v1/stream/events`（`text/event-stream`）
- `RedisBackedSseEventBus.publishLocal` 用 `event().name(topic)` 发送，**SSE `event:` 名 = 完整 topic**（如 `device.register`/`clientcmd.ptz`），`data` 为 payload 本体
- 鉴权走 URL `?token=`（EventSource 不支持自定义 header）

## 9. 测试策略（摘要）

| 层级 | 类型 | 注解 |
|------|------|------|
| Controller / Service | 纯单元 | `@ExtendWith(MockitoExtension.class)` + `@Mock`/`@InjectMocks` |
| Manager / Repository | 集成 | `@SpringBootTest` + `BaseTest`，真实库 `@Transactional` |
| HTTP API / 异步 / Hook | 集成 | `@SpringBootTest`，`TestRestTemplate`，**无事务**手动清理 |

所有测试集中在 `voglander-web/src/test/java/io/github/lunasaw/voglander`。覆盖率：`./generate-coverage-report.sh`（JaCoCo 聚合）。
