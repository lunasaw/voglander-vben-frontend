## 3. API 层

### 3.1 组织（`src/api/`）

```
api/
├── core/    auth.ts · menu.ts · user.ts          → index.ts 桶导出
├── media/   medianode.ts · stream-proxy.ts · push-proxy.ts · zlm-media.ts · zlm-query.ts
├── system/  dept.ts · menu.ts · role.ts · user.ts
├── protocol-lab.ts
├── request.ts   # 请��客户端配置
└── index.ts     # 根聚合：export * from core/media/system
```

> ⚠️ 注意：`api/media/index.ts` 当前仅桶导出 `medianode` + `stream-proxy`；`push-proxy` / `zlm-media` / `zlm-query` 由页面**按文件路径直接导入**（如 `#/api/media/push-proxy`），未走桶。

约定：请求函数从 `#/api/request` 导入 `requestClient`；类型与函数同放在该实体的 `export namespace XxxApi { ... }` 内，无独立 `types.ts`。

### 3.2 请求客户端（`request.ts`）

`requestClient`（`responseReturn:'data'`）对接后端 `AjaxResult`：

| 拦截器 | 作用 |
| --- | --- |
| 请求拦截器 | 注入 `Authorization: Bearer <token>`（来自 `accessStore`）、`Accept-Language`（`preferences.app.locale`）；URL 含 `/zlm/api` 时从 `utils/node-state.ts` 注入 `X-Node-Key` |
| `defaultResponseInterceptor` | `codeField:'code'` / `dataField:'data'` / **`successCode:0`** → 直接返回 `data` 本体 |
| `authenticateResponseInterceptor` | token 过期自动 `doRefreshToken()`（`/auth/refresh`），失败 `doReAuthenticate()` 登出/弹窗 |
| `errorMessageResponseInterceptor` | 兜底错误用 `message.error` 提示 |

`baseRequestClient`（不解包）用于 refresh/logout 等需要原始响应的场景。

## 4. 适配器层（`src/adapter/`）

统一不同 UI 库接口，提供一致开发体验：

| 文件 | 作用 |
| --- | --- |
| `component/index.ts` | 注册 Ant Design Vue 组件（异步导入）+ 自定义业务组件 `NodeSelector`；`withDefaultPlaceholder` 注入默认 placeholder |
| `form.ts` | `setupVbenForm`：`baseModelPropName:'value'`，`Checkbox/Radio/Switch`→`checked`、`Upload`→`fileList`；国际化校验规则 `required`/`selectRequired`；导出 `useVbenForm` / `VbenFormSchema` / `z` |
| `vxe-table.ts` | `setupVbenVxeTable`：全局 grid 配置（居中、可调列宽、`proxyConfig.autoLoad`）；内置 `CellSwitch`（状态切换）、Tag、Image、Popconfirm 渲染器 |

**Schema 驱动**：表单/表格行为由配置定义，不写命令式代码。

## 5. 标准 CRUD 页面结构

```
views/[module]/[entity]/
├── list.vue          # 主列表页（VXE Grid + 工具栏 + 抽屉）
├── data.ts           # useFormSchema / useColumns / useGridFormSchema
└── modules/form.vue  # 创建/编辑表单
```

`list.vue` 典型骨架：

```ts
const [Grid, gridApi] = useVbenVxeGrid({
  formOptions: { schema: useGridFormSchema() }, // 搜索表单
  gridOptions: {
    columns: useColumns(onActionClick),
    proxyConfig: { ajax: { query: getXxxPageList } }, // 分页代理
  },
});
const [FormDrawer, formDrawerApi] = useVbenDrawer({ connectedComponent: Form });
```

`data.ts` 三件套：

- `useFormSchema(isEditMode)` → 编辑表单字段（`component`/`componentProps`/`fieldName`/`label`/`rules`/`dependencies`）
- `useColumns(onActionClick, onStatusChange)` → 表格列（含操作列、`CellSwitch` 状态列）
- `useGridFormSchema()` → 顶部搜索条件

### 媒体播放器架构（组件化典范）

- **MediaPlayerManager**：中央管理器，按媒体格式自动检测并切换播放器（策略模式）
- **Hls/Flv/VideoJs Player**：具体实现，统一接口（适配器模式）
- `usePlayerDetection`：`getBestFormat` / `getPlayerForFormat`
- `shallowRef` 持有当前播放器组件，避免深响应式开销

## 7. 路由与权限

### 路由（`router/routes/modules/*.ts`）

- `media.ts`：`/media` 下 node / node/detail / list / stream-proxy / push-proxy
- `protocol-lab.ts`：`/protocol-lab/gb28181`
- `system.ts` / `dashboard.ts` / `demos.ts` / `vben.ts`
- 路由 `meta.title` 走 `$t()` 国际化；后端 `/menu/all`、`/menu/permissions` 提供动态菜单/权限路由

### 双重权限保护

```ts
// 1. 按钮可见性
{ code: 'edit', show: () => hasAccessByCodes(['Media:StreamProxy:Edit']) }
// 2. 操作时再校验
function onEdit(row) {
  if (!hasAccessByCodes(['Media:StreamProxy:Edit'])) { message.error('无权限'); return; }
}
```

权限码模式 `Module:Entity:Action`；`hasAccessByCodes` 来自 `@vben/access` 的 `useAccess()`。

## 8. 状态管理（Pinia）

| Store | 文件 | 作用 |
| --- | --- | --- |
| `authStore` | `store/auth.ts` | 登录/登出编排（配合 `@vben/stores` 的 `accessStore` 存 token） |
| `nodeStore` | `store/modules/node.ts` | 当前选中 ZLM 节点 Key（`currentNodeKey`，set/get/clear） |

**双轨节点状态**：`nodeStore`（响应式，供组件）+ `utils/node-state.ts`（模块级变量，供请求拦截器读取，无 Vue 上下文）。页面切换节点时两者同步更新，确保 `/zlm/api` 请求头携带正确 `X-Node-Key`。

## 9. i18n

- 位置 `locales/langs/{zh-CN,en-US}/{media,system,protocol-lab,page,demos}.json`

## 10. 实时事件（SSE）

`composables/useSseEvents.ts` 订阅后端事件流：

- **URL**：`${apiURL}/api/v1/stream/events?topics=<前缀域>&token=<token>`，dev 下 `apiURL=/api`，vite 代理转发 :8081
- **鉴权**：token 走 URL `?token=`（EventSource 不支持自定义 header）
- **分流**：后端 `event:` 名 = **完整 topic**（`device.register` 等），逐个 `addEventListener(topic)`；订阅参数用**前缀域**（`device,session,clientcmd,alarm`）缩短 URL，完整 topic 列表由 `/lab/client/config` 提供
- **去重**：按 `(topic|ts|业务id)` 去重，规避单机 Redis 回环双投
- **方向**：`clientcmd.*` = 平台→设备（`in`）；其余 `device.*`/`session.*`/`alarm.*` = 设备→平台（`out`）→ 渲染为 SIP 时间线左右两侧
- **重连**：`onerror` 后 3s 自动重连；事件用 `shallowRef` 整体替换，上限 500 条防内存增长
